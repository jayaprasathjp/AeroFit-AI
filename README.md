# AeroFit Resolver

A Retrieval-Augmented Generation (RAG) app that lets aircraft mechanics search
Boeing 747-8F maintenance manuals for approved alternate parts.

## Structure

```
/backend
  /data        (place mock_747_amm.pdf here)
  /chroma_db   (generated by ingest.py)
  main.py
  ingest.py
  requirements.txt
/frontend
  /public      (place a copy of mock_747_amm.pdf here)
  /src
    App.jsx
    /components
      ChatPanel.jsx
      PdfViewer.jsx
```

## Backend

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate        # Windows (PowerShell: .venv\Scripts\Activate.ps1)
pip install -r requirements.txt

# 1. Put mock_747_amm.pdf in backend/data/
python ingest.py              # builds the local ChromaDB vector store

# 2. (Optional) configure Vertex AI
copy .env.example .env        # then fill in GCP_PROJECT etc.

# 3. Run the API
uvicorn main:app --reload --port 8000
```

The `/api/search` endpoint accepts `{"query": "..."}` and returns
`{"answer": "...", "page": <int>}`. If Vertex AI is not configured, the endpoint
still returns the retrieved context and the correct page number.

## Frontend

```bash
cd frontend
npm install
# Put a copy of mock_747_amm.pdf in frontend/public/
npm run dev                   # http://localhost:5173
```
